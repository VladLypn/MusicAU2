# MusicAU Landing Page

Static landing page plus waitlist APIs for local development and Vercel.

## Run Locally

```bash
python3 server.py
```

Open:

```text
http://127.0.0.1:4173/
```

Local submissions are stored in `waitlist.db` by default.

## API

- `GET /api/waitlist` returns the current waitlist count.
- `POST /api/waitlist` stores a waitlist submission.
- `GET /api/waitlist/count` is kept for local backwards compatibility.

Each submission stores:

- email
- optional contact methods
- referral source
- feature request
- created and updated timestamps

## Environment Variables

- `PORT` or `MUSICAU_PORT`: server port, default `4173`
- `MUSICAU_HOST`: host, default `127.0.0.1`
- `MUSICAU_DB_PATH`: SQLite database path, default `./waitlist.db`
- `MUSICAU_RATE_LIMIT_MAX`: submissions per IP window, default `5`
- `MUSICAU_RATE_WINDOW_SECONDS`: rate window, default `600`
- `MUSICAU_IP_HASH_SALT`: salt used when hashing IP addresses

## Deploy

### Vercel

The static site will deploy on Vercel. The waitlist backend uses
`api/waitlist.py`, a Python Vercel Function. It requires Supabase for persistent
storage because Vercel Functions cannot use a local SQLite file as a durable
production database.

Create this table in Supabase:

```sql
create table if not exists waitlist_submissions (
  id bigint generated by default as identity primary key,
  email text not null unique,
  contact_methods jsonb not null default '[]'::jsonb,
  referral_source text,
  feature_request text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  ip_hash text
);
```

Then add these Vercel environment variables:

- `SUPABASE_URL`: your Supabase project URL
- `SUPABASE_SERVICE_ROLE_KEY`: server-side Supabase service role key
- `MUSICAU_IP_HASH_SALT`: any long random string
- `MUSICAU_RATE_LIMIT_MAX`: optional, default `5`
- `MUSICAU_RATE_WINDOW_SECONDS`: optional, default `600`

Deploy from GitHub by importing this repo in Vercel. No build command is needed.

### VPS

For a small VPS, run `python3 server.py` behind Nginx/Caddy with HTTPS and set
`MUSICAU_HOST=0.0.0.0`. For production, use a persistent database location via
`MUSICAU_DB_PATH` and set a private `MUSICAU_IP_HASH_SALT`.

The backend validates and sanitizes inputs, includes a honeypot field, rate
limits by IP, and returns JSON success/error responses.
